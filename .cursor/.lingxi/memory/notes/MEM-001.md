## When to load

- 编写或评审给 Agent 的指令（如 Skill、Command、规则）时
- 发现指令被写成「第一步、第二步」式步骤清单，且希望提升执行效果时
- 需要节省 prompt 并减少错误过程建议时

## One-liner (for injection)

设计 Agent 指令时优先用「以问题/目标为导向」的表述（如「带着 N 个问题理解 X」），少用面向过程的步骤说明，以更好发挥 Agent 理解与主动解决问题能力、节省 prompt，并避免错误过程建议导致执行变差。

## Context / Decision

- **Decision**: 指令应引导 Agent 理解问题与目标并自行拆解步骤，而非替 Agent 规定具体步骤顺序。
- **Signals**: 表述中是否以「要解决什么问题/达成什么目标」「带着哪些问题去看」为主；若以「先做 A 再做 B」为主则为过程导向。
- **Alternatives**: 面向过程的步骤说明（易导致 prompt 膨胀、步骤与场景不匹配时执行效果差）。
- **Counter-signals**: 若场景强依赖确定流程（如合规检查清单），可保留必要步骤，但仍可先写「目标/问题」再补步骤。
