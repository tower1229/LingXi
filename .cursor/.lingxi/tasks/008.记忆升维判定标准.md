# 记忆升维判定标准（008）

> 与 `008.req.记忆升维治理.md`、`008.iteration-outline.品味识别与沉淀.md` §6.5.6 一致；lingxi-memory 在映射生成 note 后、写入前按本标准与评分卡执行判定。

## 1. 低价值记忆定义

**低价值记忆**：真但无法显著提升未来决策质量的条目。表现为：

- 仅复述单次事实、单对象/单文件细节，难以迁移到同类场景；
- 未来难以被检索命中（触发性弱）；
- 纯主观、不可验证或高时效、一次性易过期。

判定口径：008.req §12 五维评分卡总分 T≤3 时视为低价值，**不写**，避免“正确废话”入库。

## 2. L0 / L1 决策规则

- **L0（实例事实层）**：在具体场景下发生的可验证事实/操作与结果；用于保留证据，不强行抽象。适用：T∈[4,5] 且 D4≥1，或评分卡判定为“写事实”。
- **L1（可复用原则层）**：在场景族中优先某策略、避免某反策略；用于指导未来同类决策。适用：T∈[6,7] 且 D1+D2≥3，或 T≥8 且 D2≥1 且 D4≥1 时的原则部分。
- **双层（L0+L1）**：同时保留证据层与可迁移原则层。适用：T≥8 且 D2≥1 且 D4≥1。

是否写 L0、L1 或双层由 **008.req §12.3** 自动判定规则决定；payload 映射仅保证含足够信息可生成二者（见迭代大纲 §6.5.6）。

## 3. 书写模板（执行时套用）

- **L0 一句话**：在 `[具体场景]` 下，发生了 `[可验证事实/操作]`，导致 `[结果]`。（008.req §12.5）
- **L1 一句话**：在 `[场景族]` 中，优先 `[策略]`，避免 `[反策略]`，因为 `[目标函数/风险]`。（008.req §12.5）

与 payload→note 映射一致：One-liner 偏「做什么/不做什么」，When to load 偏「何时加载」。

## 4. 例外条件

- **D4=0**（不可验证）：即使总分高也不直接写 L1；降级为“不写”或“L0 待确认”。（008.req §12.4）
- **D5=0 且一次性解决**：默认不写；除非 D1=2 且 D2≥1，可写 L1 弱版本并标注适用边界。
- **存在明显反例或冲突记忆**：优先进入门控（merge/replace/veto），不走静默写入。

## 5. 生命周期与 Status

- **active**：当前有效，参与检索与注入。
- **local**：降级为仅本机/本地可见，或低优先级。
- **archive**：归档，不再参与默认检索；可被 Supersedes 取代或长期未命中后迁移。

触发条件、迁移节奏与回退策略见 008.req 生命周期治理规则（F4）及 memory-system 相关约定。
