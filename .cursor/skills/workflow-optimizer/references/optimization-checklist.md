# 调优检查清单

## 概述

基于项目的核心价值指引（Why 和 How）和技术能力边界，提供调优检查清单。

## 价值对齐检查（基于 Why 和 How）

### 心有灵犀

- [ ] **是否有助于经验沉淀和复用？**（有助于记住创造者的指引）
- [ ] **是否有助于让 AI 按创造者的方式做事？**
- [ ] **是否有助于项目级记忆能力？**

### AI Native

- [ ] **是否尊重 AI 作为能力源泉？**
- [ ] **是否为 AI 进化预留空间？**
- [ ] **是否过度约束 AI？**（否 → 通过，是 → 需要调整）

### 称心如意

- [ ] **是否降低认知负担？**
- [ ] **是否提供友好的交互体验？**
- [ ] **是否让常用操作更便捷？**

## 技术能力边界检查

### 组件选择

- [ ] **选择的组件是否符合其官方定义的适用场景？**
- [ ] **选择的组件是否在其能力边界内？**
- [ ] **是否有更适合的组件选择？**

### 性能考虑

- [ ] **性能开销是否可接受？**（基于官方定义的限制）
- [ ] **是否有性能瓶颈？**（否 → 通过，是 → 需要优化）

## 调优建议输出

基于检查清单，输出：

1. **通过项**：哪些检查项已通过
2. **需要调整项**：哪些检查项需要调整，如何调整（参考 Why 和 How）
3. **技术边界问题**：哪些超出技术能力边界，如何调整
4. **风险评估**：哪些检查项存在风险，如何缓解
