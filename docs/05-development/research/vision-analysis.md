# 远景纲领：保护创造者的判断力、品味与责任感

> 这是一份**长期纲领性文档**：当工具形态、模型能力不断变化时，它回答"创造者最宝贵的是什么"。
>
> 它的目的不是描述我们做了什么，而是锚定我们**不能牺牲什么**。

---

## 1. 判断力、品味与责任感（优秀创造者的核心价值）

优秀创造者最宝贵的不是“能写多少代码”，而是内在的三种素养——它们共同构成创造者的**决策与行动操作系统**：

- **判断力**：在不确定性中做取舍，定义问题、选择方向、决定何时停。
- **品味**：对“好/优雅/恰当”的内在标准，决定产出的上限与长期可维护性。
- **责任感**：对创造物的影响主动负责，包含对用户、安全、团队、长期成本的承担。

灵犀的远景以此为核心：**保护这三者的稀缺性，并让它们在长期实践中持续增强、可复利。**

---

## 2. AI 时代的隐性风险（稀缺性、脆弱性与不可复用）

大模型拥有强到超出人类的能力（检索、生成、覆盖面、速度）——这是**魄**；但魄没有主观能动性，只能被动应用，必须被创造者的“魂”（判断力/品味/责任感）引导。AI 时代的隐性风险，正发生在“魂”被稀释与被绕过之时：

- **判断力更稀缺**：速度提升让人更容易跳过问题界定与取舍，复杂性被掩盖而非被管理。
- **品味更脆弱**：产出更快更“像样”会把标准拉低到“能用即可”，让卓越标准被磨平。
- **责任感更易漂移**：当产出来自人机协作，责任边界模糊，容易出现“没人真正拍板”的空心决策。

与此同时，现实处境是：这些关键素养**难以沉淀复用**——判断的理由、拒绝的方案、接受的风险、验证的方法往往停留在一次对话里，导致：

- 同类问题反复发生（不复利）
- 建议与决策不稳定（易漂移）
- 经验库被噪音污染或过时（难治理）

这就是灵犀要解决的核心矛盾：**让魂能稳定地引导魄，并把“魂的取舍”变成可复用资产。**

---

## 3. 灵犀——创造者的专属法宝（为什么能解决问题）

灵犀不是“更聪明的聊天”，而是一套为创造者打造的工作流系统：**让魂（判断力/品味/责任感）有效引导魄（大模型能力），并把这种引导沉淀为可维护、可治理、可复利的资产。**

灵犀之所以能解决问题，是因为它把“人机协作”拆成两层并分别工程化：

- **控制面（先保证魂在场）**：关键节点必须由人确认，避免魄在没有魂引导下自动推进。
- **资产面（让魂可复利）**：把取舍依据、经验结论、质量标准固化为长期资产，并且可治理、可升级。

为更好服务创造者，灵犀采用以下关键设计（它们的共同目标：保护魂、稳定引导、避免漂移）：

- **质量资产化（Quality-as-Assets）**：把模板/清单/评审量表/写作规范等“质量表达”下沉为可维护资产，让质量“可执行”（载体可以是 Skills / Rules / Hooks / CI，且可逐步升级）。
- **单入口 `/flow`**：统一协议与体验，减少分裂与绕行。
- **人工闸门（Human Gates）**：阶段推进、规则采纳、经验写入都必须明确确认，责任边界清晰。
- **确认式沉淀（Confirm-only）**：机器负责不漏、人负责不滥，避免低质量沉淀污染质量资产。
- **最小高信号上下文**：指针优先、按需加载，防止噪音稀释判断力。
- **即时捕获 + 分层处理**：在工作过程中捕获候选（如 EXP-CANDIDATE），先暂存、再确认、再进入长期经验库。
- **经验治理与升级**：合并/取代保持经验库精炼，把高频判断升级为质量规则（qs-*）乃至更强门控（Hook/CI）。
- **SSoT 索引与可审计**：用索引与产物结构保证“什么时候、谁、以何依据做了什么”可追溯。

一句话：**灵犀让魂可见、可控、可沉淀；让魄强大但受约束；魂魄结合，才有稳定造物。**

## 4. 当前阶段（事实表述）

当前灵犀已具备“让魂引导魄”的骨架，但机制尚需打磨：

**已具备**：

- 单入口、阶段路由、人工闸门（魂控制魄）
- 确认式沉淀、最小高信号上下文（魂的引导被记录）
- 经验候选收集与治理的基本形态（魂的沉淀）

**尚需强化**：

- 取舍依据与价值取向的显式捕获（让灵犀知道魂的"为什么"）
- 反馈闭环（魂的引导是否真的有效）
- 可观测性与审计（魄的运行时，魂的引导是否被遵循）

---

## 5. 路线图：按“魂引导魄”的成熟度推进

### Phase A（P0）：纪律底盘

目标：把"魂控制魄"的机制做到足够强，让灵犀可被长期信任。

- 强化闸门协议：每个阶段的"可推进判据"更清晰，确保魄的运行必须经过魂的确认。
- 沉淀质量标准化：什么该沉淀、什么不该沉淀，确保只有真正的魂被记录。
- 治理策略落地：合并/取代/升级的触发条件与流程，让魂的演进可控。

### Phase B（P0）：价值取向捕获

目标：从"知道魄做了什么"升级为"知道魂为什么这样引导魄"。

- 显式捕获取舍依据：trade-off、风险接受、拒绝了什么——这些都是魂的体现。
- 业务语义抽象：把业务边界/规则从需求中抽取为长期上下文，让魂的引导更精准。

### Phase C（P0）：反馈闭环

目标：让灵犀对"魂的引导是否有效"负责。

- 使用反馈追踪：魂的引导被加载/被采用/被忽略。
- 有效性评估与淘汰：无效的魂可降权、可归档。

### Phase D（P1）：主动性

目标：在可控前提下，让魄更主动地运用沉淀的魂。

- 风险预警：基于历史失败模式（魂的判断）提前提醒。
- 关联推荐：基于经验网络（魂的沉淀）进行建议，但必须可解释、可关闭。

### Phase E（P2）：跨项目传承

目标：让创造者的魂跨项目复利。

- 可迁移资产分层：区分项目特有 vs 通用判断（魂的层次）。
- 迁移治理：迁移后的魂需验证与本地化，确保魄的引导在新项目中依然有效。
