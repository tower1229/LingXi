# 远景目标分析与当前阶段评估

## 远景目标

为超级个体打造一个**灵犀法宝**，具备两个核心能力：

1. **需求提纯并放大**：与传统 AI 工作流一致，帮助用户更好地表达和执行需求
2. **沉淀抽象知识**：随着使用逐渐沉淀开发经验、价值取向、业务特性等抽象知识，从而越来越与使用者心有灵犀，越来越懂得项目的开发套路

## 当前阶段评估

### 已实现的核心能力（MVP 阶段）

#### 1. 需求提纯与放大 ✅
- **工作流生命周期管理**：req → plan → audit → work → review → archive
- **外部知识放大**：req/plan 阶段使用 WebSearch/MCP 获取最佳实践
- **Skills-first 架构**：高质量模板、检查清单、写入规范
- **人工闸门**：过程可控，质量可保证

#### 2. 经验沉淀基础 ✅
- **即时捕获机制**：EXP-CANDIDATE + experience-collector（后台静默收集）
- **成长过滤器**：区分短期项目记录和长期判断资产
- **经验治理**：合并/取代，保持经验库精炼
- **质量准则提炼**：从经验到规则（qs-*）

#### 3. 基础设施 ✅
- **上下文工程**：最小高信号上下文原则
- **SSoT 机制**：INDEX.md 作为单一事实源
- **确认式沉淀**：人工闸门保证质量
- **目录结构**：已预留 business/ 目录

### 当前阶段定位

**阶段**：**MVP → 成长阶段过渡期**

- ✅ **MVP 已完成**：核心机制已实现，可以开始使用和积累经验
- 🚧 **成长能力初具雏形**：经验捕获和治理机制已建立，但"心有灵犀"能力还在初级阶段

### 与远景目标的差距分析

#### 差距 1：价值取向捕获不足

**现状**：
- 只捕获"技术判断"（Decision Shape + Judgment Capsule）
- 没有专门捕获"价值取向"（为什么选择这个方案，背后的价值观是什么）

**缺失**：
- 价值取向的显式字段（如：`value-orientation`、`trade-off-preference`）
- 价值取向的沉淀机制
- 价值取向的应用机制（在决策时考虑用户的价值偏好）

**影响**：
- 系统无法理解用户"为什么这样选择"，只能理解"选择了什么"
- 无法在相似场景下主动推荐符合用户价值取向的方案

#### 差距 2：业务特性抽象机制缺失

**现状**：
- `business/` 目录已预留，但为空
- 没有业务特性捕获和抽象的机制

**缺失**：
- 业务特性的识别机制（从对话/需求中识别业务概念）
- 业务特性的抽象模板（业务边界、协作模式、业务规则）
- 业务特性的应用机制（在相关场景自动加载）

**影响**：
- 无法沉淀项目特有的业务知识
- 无法在跨项目时复用业务理解

#### 差距 3：个人偏好学习缺失

**现状**：
- 没有跟踪用户的选择模式
- 没有跟踪用户的拒绝模式
- 没有跟踪用户的偏好模式

**缺失**：
- 选择模式分析（用户倾向于选择哪些方案）
- 拒绝模式分析（用户倾向于拒绝哪些方案）
- 偏好模式学习（从多次选择中学习偏好）

**影响**：
- 无法个性化适配
- 无法主动推荐符合用户偏好的方案

#### 差距 4：使用反馈循环缺失

**现状**：
- 经验被匹配和提醒，但没有跟踪"是否被使用"
- 没有跟踪"哪些经验有用，哪些没用"

**缺失**：
- 经验使用跟踪（哪些经验被加载、被应用）
- 经验有效性评估（哪些经验帮助了用户，哪些被忽略）
- 经验权重调整（根据使用情况调整匹配优先级）

**影响**：
- 无法优化经验匹配
- 无法淘汰无效经验
- 无法提升"心有灵犀"的准确性

#### 差距 5：跨项目知识迁移机制缺失

**现状**：
- 成长过滤器提到"跨项目"，但没有跨项目的知识共享机制
- 经验库是项目级的，无法跨项目复用

**缺失**：
- 跨项目知识库（用户级或全局级）
- 跨项目知识迁移机制
- 跨项目知识应用机制

**影响**：
- 无法在多个项目间共享知识
- 无法形成"个人知识库"

#### 差距 6：主动建议能力不足

**现状**：
- 只在匹配到经验时提醒
- 没有主动预测和推荐

**缺失**：
- 意图预测（预测用户下一步要做什么）
- 主动建议（在用户还没问之前就给出建议）
- 风险预警（提前识别潜在风险）

**影响**：
- 无法实现真正的"心有灵犀"
- 只能被动响应，不能主动帮助

#### 差距 7：知识关联分析缺失

**现状**：
- 经验之间只有简单的谱系关系（Replaces/ReplacedBy）
- 没有更深层的关联分析

**缺失**：
- 经验关联图谱（哪些经验经常一起出现）
- 场景关联分析（哪些场景需要哪些经验的组合）
- 知识网络构建

**影响**：
- 无法形成知识网络
- 无法进行关联推荐

## 未来工作规划

### Phase 1: 价值取向与业务特性捕获（优先级：P0）

**目标**：让系统能够捕获和理解用户的价值取向和业务特性

#### 1.1 价值取向捕获机制

**新增字段**：
- 在经验模板中新增 `Value Orientation` 字段
- 在 INDEX 中新增 `Value Tags` 字段（如：`performance-first`、`simplicity-first`、`security-first`）

**捕获机制**：
- 在 Decision Shape 中显式捕获价值权衡
- 在 Judgment Capsule 中捕获价值取向的演变
- 从用户的选择模式中推断价值取向

**应用机制**：
- 在方案选择时，优先推荐符合用户价值取向的方案
- 在经验匹配时，考虑价值取向的匹配度

#### 1.2 业务特性抽象机制

**识别机制**：
- 从需求描述中识别业务概念（实体、流程、规则）
- 从对话中识别业务边界和协作模式

**抽象模板**：
- 业务实体模型（`.workflow/context/business/entities/`）
- 业务流程模型（`.workflow/context/business/processes/`）
- 业务规则模型（`.workflow/context/business/rules/`）

**应用机制**：
- 在相关场景自动加载业务上下文
- 在跨项目时复用业务理解

### Phase 2: 个人偏好学习与反馈循环（优先级：P0）

**目标**：让系统能够学习和适应用户的个人偏好

#### 2.1 选择模式分析

**跟踪机制**：
- 跟踪用户在方案选择时的选择
- 跟踪用户对质量准则建议的采纳/拒绝
- 跟踪用户对经验提醒的响应

**分析机制**：
- 分析用户的选择模式（倾向于选择哪些类型的方案）
- 分析用户的拒绝模式（倾向于拒绝哪些类型的方案）
- 构建用户偏好模型

#### 2.2 使用反馈循环

**跟踪机制**：
- 跟踪经验的使用情况（被加载、被应用、被忽略）
- 跟踪经验的有效性（是否帮助了用户）

**优化机制**：
- 根据使用情况调整经验权重
- 根据有效性淘汰无效经验
- 根据反馈优化经验匹配算法

### Phase 3: 主动建议与意图预测（优先级：P1）

**目标**：让系统能够主动预测和推荐

#### 3.1 意图预测

**预测机制**：
- 基于当前上下文预测用户下一步要做什么
- 基于历史模式预测用户意图

**应用机制**：
- 提前加载相关经验
- 提前准备相关上下文

#### 3.2 主动建议

**建议机制**：
- 在用户还没问之前就给出建议
- 基于风险预警主动提醒

**应用机制**：
- 在关键决策点主动建议
- 在潜在风险点主动预警

### Phase 4: 知识关联与网络构建（优先级：P1）

**目标**：构建知识网络，实现关联推荐

#### 4.1 经验关联分析

**分析机制**：
- 分析哪些经验经常一起出现
- 分析哪些场景需要哪些经验的组合

**应用机制**：
- 构建经验关联图谱
- 实现关联推荐

#### 4.2 知识网络构建

**构建机制**：
- 构建经验-场景-业务的知识网络
- 实现知识图谱可视化

**应用机制**：
- 基于知识网络进行关联推荐
- 基于知识网络进行场景理解

### Phase 5: 跨项目知识迁移（优先级：P2）

**目标**：实现跨项目的知识共享

#### 5.1 用户级知识库

**机制**：
- 在用户级别维护知识库（而非项目级别）
- 实现跨项目的知识共享

**应用机制**：
- 在新项目中自动加载相关经验
- 在新项目中自动加载相关业务理解

#### 5.2 知识迁移机制

**机制**：
- 识别可迁移的知识（跨项目通用）
- 实现知识的导入/导出

**应用机制**：
- 在新项目中导入相关经验
- 在新项目中导入相关业务理解
