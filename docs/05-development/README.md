## 使用指南

### 何时创建调查备忘

在以下情况下创建调查备忘文档：

1. **技术调研**：需要深入分析某个技术、工具或框架的能力边界
2. **架构评估**：需要评估当前架构的合理性，寻找优化空间
3. **阶段评估**：需要评估项目当前阶段，规划未来工作
4. **问题分析**：需要深入分析某个问题的根本原因和解决方案

### 调查备忘文档格式

建议包含以下部分：

```markdown
# [标题]

## 背景
为什么需要这个调查

## 调查内容
详细的调查过程和分析

## 结论
调查得出的结论和建议

## 后续行动
基于调查结果需要采取的行动
```

### 开发任务格式

建议包含以下部分：

```markdown
# [任务标题]

## 任务描述
任务的具体描述

## 状态
- [ ] 待开始
- [ ] 进行中
- [ ] 已完成
- [ ] 已取消

## 优先级
- 高 / 中 / 低

## 相关资源
- 相关文档链接
- 相关 issue 链接
```

## 维护原则

- 调查备忘文档应该保持独立，不依赖其他文档
- 开发任务应该及时更新状态
- 定期清理过时的文档和任务
