# 开发任务

> 记录项目开发过程中的具体任务和 issue。

## 任务分类

### 待办任务

记录需要完成但尚未开始的任务。

### 进行中任务

记录正在进行的任务。

### 已完成任务

记录已完成的任务（可定期归档）。

### 已取消任务

记录已取消的任务及取消原因。

## 任务格式

建议使用以下格式：

```markdown
# [任务标题]

## 任务描述
任务的具体描述和目标

## 状态
- [ ] 待开始
- [ ] 进行中
- [ ] 已完成
- [ ] 已取消

## 优先级
- 高 / 中 / 低

## 相关资源
- 相关文档链接
- 相关 issue 链接
- 相关代码位置

## 备注
其他需要记录的信息
```

## 维护原则

- 任务应该及时更新状态
- 已完成的任务可以定期归档
- 已取消的任务应该记录取消原因
