# REQ-006: 优化 skills-creator 和 rules-creator，添加创建日期元数据

| 属性     | 值       |
| -------- | -------- |
| 版本     | 1.0      |
| 状态     | 草稿     |
| 创建日期 | 2026-01-15 |
| 需求类型 | 简单功能 |
| 复杂度   | 简单     |

---

## 1. 概述

### 1.1 背景

当前 `skill-creator` 的 `init_skill.py` 脚本在创建 SKILL.md 时，模板中包含 `{DATE}` 占位符但未填充真实日期。`rules-creator` 创建规则时也未在 frontmatter 中包含创建日期元数据。为了便于追踪和管理，需要为两者添加创建日期元数据。

### 1.2 问题描述

1. **skill-creator**：`init_skill.py` 生成的 SKILL.md 模板中 `{DATE}` 占位符未被替换为真实日期
2. **rules-creator**：创建的规则文件缺少创建日期元数据，仅在文件末尾有 Source/Adopted 日期
3. **最佳实践对齐**：需要参考 AgentSkills 规范和 skill-creator 的最佳实践优化 rules-creator 的实现

### 1.3 解决方案概述

1. 修改 `init_skill.py`：调用系统命令获取当前日期，替换模板中的 `{DATE}` 占位符
2. 优化 `rules-creator`：
   - 在创建规则时调用系统命令获取创建日期
   - 在 frontmatter 的 `metadata` 字段中添加 `created` 日期（符合 AgentSkills 规范）
   - 参考 skill-creator 的结构和最佳实践优化实现
3. 确保日期格式统一：使用 `YYYY-MM-DD` 格式

---

## 2. 目标

| 编号 | 目标 | 优先级   |
| ---- | ---- | -------- |
| G1   | skill-creator 生成的 SKILL.md 包含真实的创建日期 | 必须实现 |
| G2   | rules-creator 创建的规则文件 frontmatter 包含创建日期元数据 | 必须实现 |
| G3   | rules-creator 实现符合 AgentSkills 规范和最佳实践 | 必须实现 |

---

## 3. 验收检查清单

- [ ] `init_skill.py` 生成的 SKILL.md 中 `{DATE}` 被替换为真实日期（格式：YYYY-MM-DD）
- [ ] `rules-creator` 创建的规则文件 frontmatter 包含 `metadata.created` 字段
- [ ] 创建日期通过系统命令获取（`date +%Y-%m-%d` 或 Python `datetime`）
- [ ] rules-creator 的实现结构参考 skill-creator 的最佳实践
- [ ] 所有日期格式统一为 `YYYY-MM-DD`
- [ ] 验证脚本和打包脚本正常工作（如有影响）
